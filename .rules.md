## ⚠️ GOLDEN RULE — Trust Kalshi's live WS feed

**Never compute, cache, or REST-poll data that Kalshi's WebSocket
provides.** The `ticker` channel sends `yes_bid_dollars` and
`yes_ask_dollars` for every market on every change. Use them.

If you're about to compute a mid-price, cache a value, or make a
REST call for market data — stop. Check the WS feed first.

## Project Context

This is a live Python trading bot that interacts with the Kalshi prediction
markets API. It places real orders with real money. The primary entry point
is `main.py`.

Key project files to be aware of:
- `main.py` — bot entry point (see Bot Startup Policy below)
- `ai/handoff.md` — read before every session
- `ai/decisions.md` — read when touching architecture
- `ai/kalshi-knowledge.md` — read fully before any Kalshi-related work
- `ai/KALSHIMASTERAPIINDEX.md` — Kalshi API reference

---

## Bot Startup Policy ⚠️

**Only the user may start `main.py`.** Never start, restart, or launch
`main.py` without explicit user permission for that specific occasion.

**Why:** Two simultaneous instances place duplicate live orders. This costs
real money and cannot be undone.

- Your role with a running bot: tail logs, monitor output, report issues.
- If the bot needs starting, ask and wait for an explicit yes.
- This rule is never overridden by any other instruction.

## Terminal Commands

This project runs on Windows. The default shell is `sh` (via PowerShell).
Both bash-style and PowerShell commands work in the terminal tool. Use
whichever is more natural for the task.

Common commands that work:
- `tail -f file.log` — follow a log file
- `grep "pattern" file` — search for text
- `git add/commit/push` — version control
- `python -c "code"` — run inline Python
- `cd dir && command` — change dir and run command

For PowerShell-specific operations, use `pwsh -c "command"`.
